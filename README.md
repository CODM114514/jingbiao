# 竞价网站（在线拍卖）

这是一个轻量的 Node.js 小网站：

- 竞标人注册 / 登录（含密码和安全问题）
- 到开拍时间前显示倒计时，开拍后可实时加价
- 支持公开递增竞价，实时看到其他人手机号后 4 位
- 管理员可在后台设置开拍时间、起拍价、加价步长

## 本地运行

```bash
npm install
npm run start
```

默认访问：

- 用户页：`http://127.0.0.1:3000/`
- 管理页：`http://127.0.0.1:3000/admin.html`

端口可通过 `PORT` 环境变量调整。

## 一键托管到 Render（你说的“直接上网”）

项目已内置 Render 配置：`render.yaml`（包含启动命令和基础环境变量）。

### 最快流程

1. 把仓库推到 GitHub（`main` 分支）。
2. 在 Render 新建 Web Service，直接从 GitHub 导入仓库。
3. 按以下设置保存：
   - Runtime：Node
   - Build Command：`npm install`
   - Start Command：`npm run start`
   - Plan：Free
   - Health Check：`/api/public`
4. 环境变量：  
   - `HOST=0.0.0.0`
   - `ADMIN_PASSWORD=<你自定义密码>`
   -（可选）`DATA_FILE=<自定义路径>`，不配则默认 `data/auction.json`
   
   如果你在创建页面**看不到** Environment 区域，别担心：  
   这是蓝图（render.yaml）导入的常见界面差异。你可以直接在 `render.yaml` 里改好 `ADMIN_PASSWORD`，推送到 GitHub 后让 Render 重新部署即可。
5. 点击 `Create Web Service`。
6. 部署完成后会给你一个公开网址，例如：  
   `https://xxx.onrender.com`
7. 用户入口：`/`，管理员入口：`/admin.html`

### Render 一键部署按钮（可放到你的 README 里）

```md
[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/<你的用户名>/<你的仓库名>)
```

把链接里的 `<你的用户名>/<你的仓库名>` 换成真实仓库地址即可。

## 注意（免费套餐）

- Render 免费实例会有空闲休眠（第一次访问可能有冷启动延迟）。  
- 免费实例文件系统是**临时**的，数据以本地 JSON 文件保存，重启/重建后可能重置。若你要长期留存建议后续接数据库。

## 目录结构（便于你快速确认）

- `server.mjs`：后端 API 与静态页服务
- `src/auctionCore.mjs`：竞价核心规则
- `src/store.mjs`：JSON 数据存储
- `public/index.html`：竞标页
- `public/admin.html`：后台管理页
- `public/app.js`、`public/admin.js`：前端逻辑
- `test/`：核心规则与 API 测试
- `render.yaml`：Render 部署配置

## 你选的大陆托管方式：腾讯云 CloudBase（适合国内访问）

### 适用场景

熟人内测（5~10 人）、偶尔访问、先快速上线。

### 适合小白的最短流程（3 分钟）

#### 第 1 步：先进控制台

1. 打开 `https://tcb.cloud.tencent.com/` 登录，完成授权后进入控制台首页。
2. 在左上角选择（或创建）一个环境。  
   新用户通常会有免费的个人版体验环境可用（官方说明中叫“免费体验环境”）。

#### 第 2 步：新建云托管服务

1. 左侧菜单点 **云托管**。
2. 点 **服务列表 / 新建服务**。
3. 选择 **Git 仓库部署**（或者 **通过公开 Git 仓库地址部署**）。
4. 代码仓库填你的 GitHub 地址，分支填 `main`。
5. 端口填 `3000`（你的应用默认监听这个端口）。
6. 点击 `保存并部署`。

#### 第 3 步：设置管理员密码

1. 服务创建成功后，点服务名进入详情。
2. 在服务配置里找到环境变量（有时叫“环境变量/变量配置”）。
3. 加两条：
   - `HOST=0.0.0.0`
   - `ADMIN_PASSWORD=<你自己的管理员密码>`
4. 保存并重新部署一次即可。

#### 第 4 步：拿链接验收

1. 部署成功后打开服务默认域名（控制台会显示）。
2. 先用管理员页 `/admin.html` 试一遍登录。
3. 确认用户页 `/` 能看到倒计时和注册/登录。

默认域名可通过浏览器访问后直接用了，不用再配服务器。

### 你可能看到的字段差异（别慌）

CloudBase 的控制台版本/账号菜单会有差异，可能出现：

- 有的是叫 **新建服务**，有的是在服务列表里直接显示 **部署服务**；
- 有些页面没直接显示环境变量入口，通常在服务详情页的“服务配置”或“高级设置”里；
- 地区页面里看到支持区域提示（当前文档显示支持上海区）也不影响你用，按你选中的区域就行。

### 快速判断是否成功

- 打开用户页：`https://你的服务域名/`
- 打开后台页：`https://你的服务域名/admin.html`
- 后台页输入你设置的 `ADMIN_PASSWORD` 可以登录即成功

### 原先的快速流程（适合你已经熟手时）

1. 登录 CloudBase 控制台，创建/进入一个环境（新用户通常会有免费的个人版体验环境可用）。
2. 在环境里新建服务：

   - 代码来源选 GitHub 仓库
   - 构建命令：`npm install`
   - 启动命令：`npm run start`
3. 配置环境变量（最少）：
   - `HOST=0.0.0.0`
   - `ADMIN_PASSWORD=<你的管理员密码>`
4. 部署，拿到默认域名，先可直接访问。

- 用户页：`https://你的-cloudbase-域名/`
- 后台：`https://你的-cloudbase-域名/admin.html`

CloudBase 免费体验版偏向开发测试，域名和并发都有一定限制；你这个 5-6 人的小规模场景一般够用。生产长期建议再配数据库（不要长期依赖本地 `data/auction.json`）。
